# Изменения по записи `C.SP`

## Что сделано

- Добавлена запись параметров OVEN через Modbus `holding_register`.
- Рабочий сценарий подтвержден для `TRM138` и параметра `C.SP`.
- После записи шлюз не доверяет только ACK кадру, а перечитывает параметр и
  подтверждает успех по фактическому значению прибора.
- Добавлена защита от короткого/непрозрачного ответа на запись, который
  встречается у `TRM138`.
- Исправлено кодирование `stored_dot` для значений, пришедших из `float32`,
  чтобы двоичный хвост не ломал запись.

## Что доступно в конфигурациях

- Во всех актуальных JSON-конфигах на каждый канал публикуются:
  - `rEAd`
  - `C.SP` с `writable: true`
  - внутренний `AL.t` без публикации в Modbus
- `HYSt` в готовых конфигурациях сейчас не публикуется и не пишется.
- Схема приведена к одному виду для Windows, Linux и примеров с несколькими
  линиями и приборами.

## Проверка на реальном приборе

Проверка выполнена на `TRM138`, адрес `48`, `COM6`, `9600 8N1`.

Подтвержден сценарий:

1. прочитать текущее `C.SP`
2. уменьшить значение на `10`
3. записать обратно через Modbus TCP
4. перечитать `C.SP`
5. убедиться, что изменение видно на панели прибора
6. вернуть исходное значение

Запись и возврат значения прошли корректно.

## Практические замечания

- Для адаптера `CH344 USB-RS485` наблюдались нестабильные ответы после серии
  последовательных записей.
- После переподключения адаптера и перезапуска прибора линия снова работала
  штатно.
- Поэтому текущая реализация подтверждает запись только через `readback verify`.
